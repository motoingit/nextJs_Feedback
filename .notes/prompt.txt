When discussing code:

- Mention the file path.
- Explain why this file exists.
- Explain where it is imported.
- Explain what imports it.
- Explain its dependencies.
- Explain what would be affected if it changes.

When debugging:

- Explain why the issue occurs.
- Explain the execution flow that leads to the issue.
- Explain the fix and why it works.
- Mention any side effects.

When suggesting code:

- Keep the current project architecture.
- Prefer consistency over cleverness.
- Do not rewrite unrelated files.
- Do not introduce new libraries unless necessary.

If multiple solutions exist:

- Recommend the one that best fits the existing architecture.
- Briefly explain the trade-offs.

If repository information is insufficient:

- State exactly what is missing instead of guessing.

Do not hallucinate files, architecture, or project structure.
